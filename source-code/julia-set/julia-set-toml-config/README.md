# Julia set TOML configuration

This implementation of the Julia set extends the baseline version by reading
the run parameters from a TOML configuration file.  It uses

* the same custom 2D array implementation as the baseline;
* a serial implementation of the algorithm;
* `clap` to read the configuration file name from the command line;
* `serde` and `toml` to deserialize the configuration file.


## What is new?

In the baseline implementation, each parameter is provided as a separate
command-line argument.  In this version, the command line only selects a
configuration file, and the file provides the parameters for the run.

The configuration file contains:

```toml
max_iterations = 1000
width = 80
height = 60
c_real = -0.5125
c_imag = 0.5213
```

This makes it easier to save, edit, and reuse Julia-set runs without repeating
the full set of command-line options.


## What is it?

1. `src/main.rs` contains the main function of the application.  It parses the
   configuration file name with `clap`, reads the TOML file, deserializes it
   into a typed Rust structure, and runs the Julia-set computation.
   1. `src/matrix.rs` contains the same simple 2D array implementation used by
      the baseline version.
1. `julia-set.toml` is an example configuration file.
1. `Cargo.toml` is the manifest file for Rust's package manager, Cargo.  It
   specifies the dependencies on `clap`, `num-complex`, `serde`, and `toml`.
1. `Cargo.lock` is automatically generated by Cargo to keep track of the exact
   versions of dependencies used in the project.  It ensures that the same
   versions are used across different builds and environments.


## How to use?

To run the application with the example configuration, run:

```bash
cargo run -- julia-set.toml
```

To use a different configuration, create another TOML file with the same fields
and pass its file name instead.
