# Julia set Rayon

This implementation of the Julia set extends the baseline version by using
Rayon to compute the output matrix in parallel.  It uses

* the same custom 2D array implementation as the baseline;
* the same command-line parameters and text output format as the baseline;
* `rayon` to parallelize the independent per-grid-point iteration counts.


## What is new?

In the baseline implementation, the result matrix is computed with nested
serial loops.  In this version, the result matrix is computed from flat matrix
indices using Rayon:

```rust
let data: Vec<usize> = (0..rows * cols)
    .into_par_iter()
    .map(|index| {
        let row = index / cols;
        let col = index % cols;
        let z_value = *z.get(row, col).expect("flat index should be in bounds");
        iterate_z_value(z_value, c, max_iterations)
    })
    .collect();
```

Each grid point can be computed independently, which makes the Julia set a good
introductory example for data parallelism.


## What is it?

1. `src/main.rs` contains the main function of the application.  It parses
   command-line arguments, initializes the complex grid, computes the iteration
   counts in parallel, and prints the result matrix.
1. `src/matrix.rs` contains the simple generic 2D array implementation used to
   store the complex grid and the result matrix.
1. `Cargo.toml` specifies the dependencies on `clap`, `num-complex`, and
   `rayon`.
1. `Cargo.lock` is generated by Cargo to keep dependency versions
   reproducible.


## How to use?

Run the application and pipe the output to the visualization script:

```bash
cargo run --release -- --width 800 --height 600 | ../view-fractal.py
```

Rayon uses a thread pool by default.  To control the number of worker threads,
set the `RAYON_NUM_THREADS` environment variable:

```bash
RAYON_NUM_THREADS=4 cargo run --release -- --width 800 --height 600 | ../view-fractal.py
```

The command-line arguments are the same as for the baseline implementation, so
the output can be compared directly.


## Benchmarking thread scaling

The `benchmark.sh` script builds this implementation in release mode and runs
it with several values of `RAYON_NUM_THREADS` using `hyperfine`:

```bash
./benchmark.sh
```

The default thread counts are `1 2 4 8`.  They can be changed with the
`THREAD_COUNTS` environment variable:

```bash
THREAD_COUNTS="1 2 4 8 16" ./benchmark.sh
```

The image size and number of iterations can also be changed:

```bash
WIDTH=2400 HEIGHT=2400 MAX_ITERATIONS=1000 THREAD_COUNTS="1 2 4 8" ./benchmark.sh
```

The script writes `benchmark-results.json` by default.  Use `EXPORT_JSON` to
choose a different output file:

```bash
EXPORT_JSON=rayon-scaling.json ./benchmark.sh
```
