# Borrowing vectors

Borrowing is an importtant concept in Rust, and it is used to allow multiple
parts of a program to access the same data without taking ownership of it. This
is achieved through the use of references, which are pointers to data that do
not have ownership.

In this example a few concepts are introduced:

* vectors: a dynamic array type in Rust.
* borrowing: the act of allowing multiple parts of a program to access the same
  data without taking ownership of it.
* borrowing rules: the rules that govern how references can be used in Rust,
  including the rules for mutable and immutable references.


## What is it?

1. `src/main.rs` contains the main function of the application.  It defines
   some variables of basic types and prints them to the console.
1. `Cargo.toml` is the manifest file for Rust's package manager, Cargo.  It

1. `Cargo.lock` is automatically generated by Cargo to keep track of the exact
   versions of dependencies used in the project.  It ensures that the same
   versions are used across different builds and environments.
